# 协同捏合运动生成 — 代码实现详细分析

## 一、问题建模

捏合系统共有 **7 个关节自由度**（拇指 q1–q4 = 4 DOF + 四指 q1–q3 = 3 DOF）。捏合约束要求两指尖在三维空间中重合，即：

```
thumb_tip(q_thumb) = finger_tip(q_finger)    ← 3 个标量约束
```

7 DOF − 3 约束 = **4 维捏合流形**。也就是说，在保持指尖接触的前提下，关节空间中仍存在一个 4 维子流形可以自由运动。代码的核心任务就是**在这个流形上生成平滑轨迹**。

## 二、整体流水线

```
_find_initial_pinch()          ← 第一步：找到一个合法捏合配置
        ↓
_select_best_sweep()           ← 第二步：选择最优扫描维度
  └── sweep_joint_pinch() ×4   ← 对 q1/q2/q3/q4 各扫描一次
        ↓
generate_coordinated_trajectory()  ← 第三步：插值 + 生成完整轨迹
  ├── approach   (零位 → 首帧)
  ├── sweep      (协同运动)
  └── retract    (末帧 → 零位)
```

## 三、各阶段详解

### 3.1 `_find_initial_pinch()` — 网格搜索 + 局部优化（第 45–100 行）

**目标**：在 7 维关节空间中找到一个使两指尖尽可能接近的初始配置。

**第一阶段：网格搜索**（第 49–88 行）

```python
q1t = np.linspace(tl[0][0], tl[0][1], n_grid)       # 10 个采样点
q2t = np.linspace(tl[1][0], tl[1][1], max(n_grid//2, 4))  # 5 个
q3t = np.linspace(tl[2][0], tl[2][1], n_grid)       # 10 个
q4t = np.linspace(tl[3][0], tl[3][1], max(n_grid//2, 4))  # 5 个
```

拇指网格：10 × 5 × 10 × 5 = **2500 个样本**
四指网格：10 × 5 × 10 = **500 个样本**

每个样本执行一次 FK 计算指尖位置，然后做**分块最近对搜索**（第 79–88 行）。为避免 2500×500 的距离矩阵一次性占满内存，按 chunk=500 分批计算：

```python
for start in range(0, len(thumb_arr), chunk):
    diff = thumb_arr[start:end, None, :] - finger_arr[None, :, :]  # (chunk, 500, 3)
    dists = np.linalg.norm(diff, axis=2)                           # (chunk, 500)
```

每个 chunk 产生一个 (500, 500, 3) 的差值张量，找到该 chunk 内最小距离的索引对。

**第二阶段：局部优化**（第 90–100 行）

以网格搜索最佳对作为初始值，用 L-BFGS-B 最小化：

```python
objective(x) = ||thumb_tip(x[0:4]) - finger_tip(x[4:7])||²
```

优化变量 `x` = [thumb_q1, thumb_q2, thumb_q3, thumb_q4, finger_q1, finger_q2, finger_q3]，共 7 维。bounds 为每个关节的限位范围（从 XML 读取的度数值）。

L-BFGS-B 参数：`maxiter=500, ftol=1e-10`。这里 ftol 设得很小（1e-10），因为目标函数是距离的平方（单位 mm²），需要收敛到 <0.001mm² 才能保证接触精度。

**返回值**：优化后的拇指 4 个关节角和四指 3 个关节角（度数）。

### 3.2 `_sweep_from()` — 单方向关节扫描（第 104–151 行）

**核心思想**：固定拇指的一个关节角 `q_sweep` 为指定值，优化其余 6 个关节（拇指 3 个 + 四指 3 个）使指尖距离最小。通过逐步改变 `q_sweep` 的值，追踪捏合流形上的一条一维曲线。

**参数拆分**（第 106–113 行）：

```python
other_idx = [i for i in range(4) if i != sweep_idx]  # 拇指中除被扫描关节外的3个
```

优化变量 `x` 包含 6 个值：`[thumb_other_1, thumb_other_2, thumb_other_3, finger_q1, finger_q2, finger_q3]`

**闭包捕获技巧**（第 116 行）：

```python
def obj(x, _val=val):  # 用默认参数捕获当前 val
```

这是 Python 闭包的经典陷阱处理——如果不用 `_val=val`，所有 `obj` 函数会引用同一个 `val` 变量（循环结束后的最终值），导致优化目标错误。

**目标函数构建**（第 117–124 行）：

```python
tq = np.zeros(4)
tq[sweep_idx] = _val                    # 被扫描关节 = 固定值
for k, idx in enumerate(other_idx):
    tq[idx] = x[k]                       # 其余拇指关节 = 优化变量
fq = x[len(other_idx):]                  # 四指关节 = 优化变量
return ||thumb_tip(tq) - finger_tip(fq)||²
```

**温启动**（warm start，第 108–109, 146–147 行）：

```python
prev_tq = init_tq.copy()
prev_fq = init_fq.copy()
...
# 优化成功后：
prev_tq = tq
prev_fq = fq
```

每一步优化的初始值来自**上一步的最优解**。这是关键设计：相邻扫描点的关节角变化很小，温启动让优化器只需几步迭代就能收敛，同时保证了轨迹的关节空间连续性。

**终止条件**（第 139, 148–149 行）：

```python
if dist < 2.0:    # < 2mm → 有效捏合
    configs.append(...)
else:
    break          # 接触断裂，停止扫描
```

一旦某步优化无法维持 <2mm 的指尖距离，说明在该方向上已达到捏合可达空间的边界，立即终止扫描。这避免了在不可达区域浪费计算。

### 3.3 `sweep_joint_pinch()` — 双向扫描（第 153–183 行）

从初始捏合配置的关节值 `init_val` 出发，向限位两端分别扫描：

```python
vals_lo = np.linspace(init_val, lo, n_steps // 2 + 1)   # 向下限扫描
vals_hi = np.linspace(init_val, hi, n_steps // 2 + 1)   # 向上限扫描
```

每个方向 `n_steps//2 + 1 = 16` 个采样点。

合并时（第 179 行）：

```python
all_configs = configs_lo[::-1] + configs_hi[1:]
```

`configs_lo` 是从 init 向 lo 的顺序，反转后变成 lo→init。`configs_hi[1:]` 跳过 init（避免重复）。最终得到一条从下限到上限的连续配置序列。

### 3.4 `_select_best_sweep()` — 自动维度选择（第 191–213 行）

**动机**：不同手指的最优扫描维度不同。例如中指捏合时拇指 q1 已经在极限位（-95°），扫描 q1 只能得到 1mm 跨度；改扫描 q3 则可得到 37.6mm。

**方法**：依次尝试 4 个拇指关节（q1–q4），执行完整的 `sweep_joint_pinch()`，计算每个的接触点移动跨度：

```python
span = np.linalg.norm(pts[0] - pts[-1])  # 首尾接触点的欧氏距离
```

选择跨度最大的维度。代价是 4× 的计算时间，但换来了对任意手指配置的鲁棒性。

**实测结果**：

| 手指 | q1 跨度 | q2 跨度 | q3 跨度 | q4 跨度 | 自动选择 |
|------|--------|--------|--------|--------|---------|
| 食指 | 15.8mm | 8.8mm | **69.4mm** | 20.8mm | q3 |
| 中指 | 1.0mm | 16.3mm | **37.6mm** | 18.6mm | q3 |

### 3.5 `generate_coordinated_trajectory()` — 完整轨迹生成（第 215–311 行）

#### 3.5.1 Waypoint 提取

从扫描结果中提取所有有效捏合配置的关节角和接触点：

```python
wp_thumb  = (N_waypoints, 4)   # 拇指关节角序列
wp_finger = (N_waypoints, 3)   # 四指关节角序列
wp_contact = (N_waypoints, 3)  # 接触点位置序列
```

#### 3.5.2 Minimum-jerk 插值（第 252–283 行）

在相邻 waypoint 之间插入 `n_interp=4` 个中间帧：

```python
t = np.linspace(0, 1, n_interp + 1)[:-1]  # [0, 0.25, 0.5, 0.75]（排除1.0）
s = 10*t³ - 15*t⁴ + 6*t⁵                  # minimum-jerk 时间缩放
```

**minimum-jerk 的性质**：

| t | s(t) | 特点 |
|---|------|------|
| 0 | 0 | 起点 |
| 0.25 | 0.104 | 启动慢 |
| 0.5 | 0.5 | 中点 |
| 0.75 | 0.896 | 减速 |
| 1 | 1 | 终点 |

s(t) 的一阶和二阶导数在 t=0 和 t=1 处均为零，保证了速度和加速度的连续性。

**在关节空间中线性插值**：

```python
sweep_thumb[idx] = (1 - sj) * wp_thumb[i] + sj * wp_thumb[i+1]
```

这是关节空间插值（不是笛卡尔空间），所以中间帧的指尖不一定精确接触。每帧通过 FK 计算实际指尖距离并记录到 `distances` 数组。

**插值后精度**：waypoint 间距约 2–3°，插值后中间帧的指尖距离通常 <2mm，最大 ~1.7mm（食指 q3 扫描）。

#### 3.5.3 接近/撤回轨迹（第 288–297 行）

```python
# 接近：零位 → 首个捏合帧
approach_thumb = np.outer(s_app, sweep_thumb[0])   # (30, 4)

# 撤回：末个捏合帧 → 零位
retract_thumb = np.outer(1 - s_app, sweep_thumb[-1])  # (30, 4)
```

`np.outer(s, q)` 将标量缩放序列 `s` 与目标关节角 `q` 做外积，等价于 `q_i = s_i * q_target`。这假设起始位置为全零。

### 3.6 `map_pinch_workspace()` — 可达空间蒙特卡洛探索（第 315–364 行）

与扫描法不同，这是**无序探索**：200 次随机初始化，每次在 7 维关节空间均匀采样，然后优化最小化指尖距离。

```python
x0 = [uniform(q_min, q_max) for each of 7 joints]
minimize(||thumb_tip - finger_tip||², x0, bounds)
```

收敛到 <1mm 的配置被保留。实测 198/200 成功率，说明捏合流形在关节空间中覆盖范围很广。

**结果**：食指捏合可达空间约 30×56×49mm 的三维区域，远大于单次扫描覆盖的范围（69mm 跨度只是一维切面）。

## 四、关键设计决策与权衡

| 决策 | 选择 | 理由 | 替代方案 |
|------|------|------|---------|
| 流形探索方法 | 固定1个关节扫描 | 简单、确定性、温启动保证连续性 | 笛卡尔空间路径跟踪（需要双指IK） |
| 优化器 | L-BFGS-B | 支持 bounds 约束、拟牛顿收敛快 | SLSQP（可加等式约束）、CMA-ES（全局但慢） |
| 插值空间 | 关节空间 | 不需要额外IK求解、计算简单 | 笛卡尔空间（接触精度更高但需要每帧求解） |
| 自动维度选择 | 遍历4个关节取最大跨度 | 鲁棒、无需先验知识 | 雅可比矩阵零空间分析（理论更优雅但实现复杂） |
| 温启动 | 上一步的最优解 | 保证关节空间连续性，加速收敛 | 无温启动（可能跳到不同分支） |
| 终止条件 | dist > 2mm 立即停止 | 保守安全，避免不可达区域 | 可以尝试跳过并继续（可能发现新可达段） |

## 五、精度分析

| 阶段 | 指尖距离 | 原因 |
|------|---------|------|
| 初始捏合（优化后）| ~0.000mm | L-BFGS-B 收敛到机器精度 |
| waypoints（扫描点）| <0.5mm | 每步独立优化 |
| 插值中间帧 | <1.7mm | 关节空间线性插值不保证笛卡尔接触 |
| 接近/撤回阶段 | N/A | 指尖故意分离，从零位移动到接触位 |

如果需要更高的插值精度，可以增加 `n_sweep`（更密的 waypoint 间距）或者对插值帧执行二次优化。

## 六、计算性能

| 步骤 | 优化次数 | 每次迭代 | 总耗时（估算）|
|------|---------|---------|------------|
| 初始搜索 | 1次（7维）| 500 iter | ~0.5s |
| 单维度扫描 | 30 次（6维）| 300 iter | ~3s |
| 自动选择（4维度）| 4 × 30 = 120次 | 300 iter | ~12s |
| 蒙特卡洛探索 | 200 次（7维）| 200 iter | ~10s |

瓶颈在自动选择：需要对 4 个关节各做一次完整扫描。如果手指类型已知可以跳过（比如两个手指都选 q3）。

## 七、为什么每次扫描只用一个关节

### 7.1 维度分析

捏合流形是 4 维的（7 DOF − 3 约束）。在这个流形上运动时，我们需要选择一种参数化方式。关键问题是：**固定几个拇指关节作为扫描变量，剩余自由度由优化器处理？**

| 固定的拇指关节数 | 扫描空间维度 | 优化后得到的结构 | 实际含义 |
|:---:|:---:|:---|:---|
| 0 | 0 维 | 单个点 | 就是 `_find_initial_pinch()` 的结果 |
| **1** | **1 维** | **一条曲线** | **当前方案：沿一个关节角扫描得到一条轨迹** |
| 2 | 2 维 | 一个曲面 | 需要二维网格参数化，复杂度 O(n²) |
| 3 | 3 维 | 一个体 | 需要三维网格，复杂度 O(n³) |
| 4 | 4 维 | 整个流形 | 等价于蒙特卡洛探索，无序 |

### 7.2 固定 1 个关节的优势

选择固定 1 个关节（即 1 维扫描）是**最小可行参数化**：

1. **天然有序**：扫描值从 `lo` 到 `hi` 单调递增，生成的配置序列天然有序，可以直接作为轨迹播放
2. **温启动有效**：相邻扫描点之间只差一小步，上一步的最优解是极好的初始值，通常 3–5 次迭代即收敛
3. **连续性保证**：温启动 + 小步长确保关节空间的连续性，不会出现关节角突跳
4. **终止条件清晰**：一旦接触断裂（dist > 2mm），说明到达可达边界，直接停止

### 7.3 剩余自由度去哪了？

固定 1 个拇指关节后，剩余 6 个变量（拇指 3 + 四指 3）需要满足 3 个接触约束，理论上还有 3 个自由度未被确定。这 3 个自由度被 **L-BFGS-B 优化器隐式处理**了——优化器在满足接触约束（距离最小化）的前提下，通过温启动"惰性地"选择了最接近上一步的解，相当于在 3 维零空间中选择了一个**最小变化方向**。

这种隐式选择虽然不是最优的（未必最小化关节速度或最大化操纵度），但实践证明效果很好——关节角变化平滑，无突跳。

### 7.4 多关节扫描的替代方案

如果需要覆盖更大的工作空间（而不仅是一条曲线），可以考虑：

**方案 A：两关节网格扫描**
```
for q_i in linspace(lo_i, hi_i, N):
    for q_j in linspace(lo_j, hi_j, M):
        optimize remaining 5 DOFs for contact
```
得到 N×M 个捏合配置，形成接触面。但温启动只在内层循环有效，外层循环需要重新寻找初始值。

**方案 B：任务空间路径跟踪**
```
for p_target in cartesian_path:
    solve IK: find (thumb_q, finger_q) such that
        thumb_tip(thumb_q) ≈ finger_tip(finger_q) ≈ p_target
```
直接在笛卡尔空间定义接触点轨迹（如直线、圆弧），然后对每个目标点求双指 IK。优势是路径直观可控，劣势是 IK 求解难度大（7 变量 6 约束，冗余度仅 1）。

**方案 C：雅可比零空间运动**
利用接触约束的雅可比矩阵 J（3×7），计算零空间投影矩阵 N = I − J⁺J，在零空间中沿期望方向运动。理论最优雅，但需要解析雅可比或数值差分，实现复杂度高。

### 7.5 当前方案的合理性

对于**展示协同运动**的目标，1 维扫描已经足够：它生成了一条平滑、连续、物理合理的捏合轨迹。结合自动维度选择（选最大跨度的关节），单次扫描即可覆盖 37–69mm 的接触点移动范围，足以展示协同运动的效果。

蒙特卡洛探索（`map_pinch_workspace`）则负责展示完整的可达空间范围，两者互补。

## 八、后续优化建议与优劣取舍

### 8.1 缓存初始捏合配置

**当前问题**：`_select_best_sweep()` 调用 4 次 `sweep_joint_pinch()`，每次内部都执行 `_find_initial_pinch()`——相同的 2500×500 网格搜索 + L-BFGS-B 优化被重复 4 次。

**优化方案**：
```python
def _select_best_sweep(self, finger_name, n_sweep=30):
    init_tq, init_fq = self._find_initial_pinch(finger_name)  # 只做一次
    for si in range(4):
        configs = self._sweep_with_init(init_tq, init_fq, finger_name, si, n_sweep)
        ...
```

| | 优势 | 劣势 |
|---|---|---|
| 缓存 | 节省 ~75% 初始搜索时间（约 1.5s） | 需要拆分 `sweep_joint_pinch` 的接口 |
| 当前 | 代码简单，每次扫描完全独立 | 重复计算，总耗时 ~12s 中有 ~2s 浪费 |

**推荐**：实施缓存。改动小（<20 行），收益明确。

### 8.2 插值精度提升

**当前问题**：关节空间线性插值后中间帧的指尖距离最大 ~1.7mm，虽然在视觉上可接受，但对力控制或精密操作场景可能不够。

**优化方案 A：增加 waypoint 密度**
```python
n_sweep=60  # 从 30 增加到 60
```
Waypoint 间距从 ~3° 减至 ~1.5°，插值误差大致减半。代价是优化次数翻倍（~6s → ~12s）。

**优化方案 B：插值后二次优化**
```python
for idx in range(n_total):
    # 先做关节空间插值得到初始值
    q_init = (1 - s) * wp[i] + s * wp[i+1]
    # 再做一次快速优化修正接触
    q_opt = minimize(contact_dist, q_init, maxiter=50)
```

| | 精度 | 计算代价 | 实现难度 |
|---|---|---|---|
| 增加密度 | ~0.8mm | 2× | 改一个参数 |
| 二次优化 | <0.1mm | 3–5× | 中等（需对每帧优化） |
| 当前 | ~1.7mm | 基准 | 已实现 |

**推荐**：先尝试 `n_sweep=60`（零代码改动），不满足再加二次优化。

### 8.3 笛卡尔空间路径控制

**当前局限**：扫描关节角决定了接触点路径，但无法直接指定"接触点沿直线/圆弧移动"。

**优化方案**：
```python
def cartesian_sweep(self, finger_name, start_pos, end_pos, n_steps=30):
    path = np.linspace(start_pos, end_pos, n_steps)
    for target in path:
        # 目标：thumb_tip ≈ finger_tip ≈ target
        minimize(||thumb_tip - target||² + ||finger_tip - target||²)
```

| | 优势 | 劣势 |
|---|---|---|
| 笛卡尔路径 | 接触点轨迹完全可控，适合任务规划 | 需先知道可达空间范围，温启动效果可能变差 |
| 关节扫描（当前）| 简单鲁棒，天然保证关节连续性 | 接触点路径是关节空间的"投影"，不直观 |

**推荐**：如需与任务规划集成（如"沿桌面边缘滑动"），值得实现。纯展示场景下关节扫描已足够。

### 8.4 雅可比零空间分析

**当前方案的理论缺陷**：优化器隐式选择零空间方向，无法控制选择哪个方向。雅可比方法可以显式选择：

```python
J = numerical_jacobian(contact_constraint, q)  # 3×7
N = I - np.linalg.pinv(J) @ J                  # 7×7 零空间投影
q_dot = N @ desired_direction                   # 在零空间中沿期望方向运动
q_next = q + alpha * q_dot
```

| | 优势 | 劣势 |
|---|---|---|
| 雅可比零空间 | 理论最优雅，可控制运动方向，可最大化操纵度 | 需要数值雅可比（7×7 差分 = 14 次 FK），奇异点处不稳定 |
| 优化扫描（当前）| 实现简单，对奇异点鲁棒（优化器自动绕开）| 零空间方向不可控 |

**推荐**：学术研究或高精度需求时实现。工程展示场景下当前方案更实用。

### 8.5 多指协同扩展

当前代码处理**拇指-单指**捏合（2 指接触）。扩展到**三指捏合**（拇指同时接触食指和中指）需要：

- 约束：`thumb_tip ≈ index_tip ≈ middle_tip`（6 个约束）
- 自由度：10 DOF（拇指 4 + 食指 3 + 中指 3）
- 流形维度：10 − 6 = 4 DOF

这仍然是 4 维流形，但优化变量从 7 增加到 10，约束从 3 增加到 6。可以用相同的扫描框架，但初始搜索的网格规模会急剧增加（拇指 2500 × 食指 500 × 中指 500 = 不可行）。需要分步求解：先找拇指-食指捏合，再优化中指加入。
